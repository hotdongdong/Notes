# 命名规范

### 变量和函数命名

```
使用驼峰命名，例如 `myVariable`、`myFunction`
```

### 组件命名

```
1、普通的组件使用大驼峰命名即可，屏幕父组件在大驼峰后加Screen，例如：
`StoInformationConfirmationScreen`、`HomeScreen`

2、编写屏幕父组件时要声明组件类型，例如：
export const HomeScreen:FC<StackScreenProps<NavigatorParamList, "Home">> = () => {....}

<!-- 表示`HomeScreen`是一个函数式组件，接受的导航参数是`NavigatorParamList`类型，
并且该组件对应的路由名称是`Home` -->

```

### 样式

```
使用`类型化样式（Typed Styles）`写法编写UI样式，样式对象命名使用全大写字母和下划线，例如：

import {ViewStyle} from 'react-native';

export const INFO_TIPS: ViewStyle = {
  backgroundColor: '#FFFFFF',
  height: 65,
  paddingHorizontal: 15,
  justifyContent: 'center',
};

<!-- 使用类型化样式优势：当在样式对象中添加、删除或修改属性时，TypeScript可以即时捕获并提
供有关这些更改的错误和警告。这有助于减少运行时样式错误，提高代码的可维护性。 -->

```
